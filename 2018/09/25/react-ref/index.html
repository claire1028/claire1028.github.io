React中的几种Ref比较

React中的refs

  • 1. ref用string 来表示

React中已将它废弃, 静态类型检测不支持

  • 2.React.createRef()
    ref 是一个 特殊的属性 在react里。

    当React.createRef()应用在html dom元素上的时候，dom元素赋值给 React.createRef().current属性上。
    <Com ref={} />，当React.createRef()应用在Component上时，React.createRef().current上的值是Component的实例，所以不能将React.createRef()生成的值赋给Function Component，即无状态组件，因为他们没有实例。

  • 3.React.forwardRef()
    (1).作用在HOC组件中传递ref 到子组件中，第二参数ref传递的是组件的实例。

    (2).作用在直接访问DOM node 。

  • 4.callback Ref
    <input type="text" ref= {el => this.ref = el }/>
    也可以通过像一个props 命名可以为任何字符除了ref 传递给其他component <Com inputRef={el => this.ref = el } />，也可以在HOC中传递
